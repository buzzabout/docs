Creates an analysis against a topic. Returns `202 Accepted` immediately
with an `analysis_id` you can poll.

```http
POST /v1/analyses
```

## Request body

| Field        | Type      | Required | Notes                                                                                  |
| ------------ | --------- | -------- | -------------------------------------------------------------------------------------- |
| `topic_id`   | `string`  | yes      | Topic to analyse. Must be readable by the API key.                                     |
| `kind`       | `string`  | yes      | One of `pattern_match`, `sentiment_pulse`, `audience_profile`, `competitor_compare`.   |
| `question`   | `string`  | no       | Free-text framing the model uses to bias the analysis.                                 |
| `window`     | `object`  | no       | Time range. `{ "last": "7d" }` or `{ "from": "2026-04-01", "to": "2026-04-30" }`.      |
| `platforms`  | `array`   | no       | Subset of the topic's platforms. Defaults to all of them.                              |

## Examples

<Tabs items={['cURL', 'TypeScript', 'Python']}>
  <Tab value="cURL">
    ```bash
    curl -X POST https://api.buzzabout.ai/v1/analyses \
      -H "Authorization: Bearer $BUZZABOUT_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "topic_id": "topic_01HZXXXXXXXXXXXXXXXXXXXX",
        "kind": "pattern_match",
        "window": { "last": "7d" }
      }'
    ```
  </Tab>
  <Tab value="TypeScript">
    ```ts
    const analysis = await client.analyses.create({
      topicId: "topic_01HZXXXXXXXXXXXXXXXXXXXX",
      kind: "pattern_match",
      window: { last: "7d" },
    });
    ```
  </Tab>
  <Tab value="Python">
    ```py
    analysis = client.analyses.create(
        topic_id="topic_01HZXXXXXXXXXXXXXXXXXXXX",
        kind="pattern_match",
        window={"last": "7d"},
    )
    ```
  </Tab>
</Tabs>

## Response

```json title="202 Accepted"
{
  "id": "ana_01HZ0XXXXXXXXXXXXXXXXXXXXX",
  "status": "pending",
  "topic_id": "topic_01HZXXXXXXXXXXXXXXXXXXXX",
  "kind": "pattern_match",
  "created_at": "2026-04-28T16:08:00Z",
  "estimated_completion_seconds": 60,
  "links": {
    "self":   "/v1/analyses/ana_01HZ0XXXXXXXXXXXXXXXXXXXXX",
    "events": "/v1/analyses/ana_01HZ0XXXXXXXXXXXXXXXXXXXXX/events"
  }
}
```

Poll `links.self` until `status` is `succeeded` or `failed`. Or open
an SSE connection to `links.events` for live progress.

<Callout type="info">
  The SDKs wrap this with `analysis.wait()` so you don't have to write
  the polling loop. See the
  [first-analysis tutorial](/docs/tutorials/first-analysis).
</Callout>

## Errors

| HTTP | `error.type`        | Cause                                                        |
| ---- | ------------------- | ------------------------------------------------------------ |
| 400  | `invalid_request`   | `kind` not recognised, or `window` malformed.                |
| 403  | `forbidden`         | Key missing the `analyses:write` scope.                      |
| 404  | `not_found`         | `topic_id` doesn't exist or isn't visible to this key.       |
| 429  | `concurrency_limit` | You have too many running analyses. Wait for one to finish.  |
