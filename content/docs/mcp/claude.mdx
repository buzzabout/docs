Add the buzzabout MCP server to Claude. Once it's wired up, you can ask
Claude things like *"Find the top three hooks pulling on Reddit for
cold brew this week"* and it will call our tools, stream the result
back, and cite the mentions it found.

## Prerequisites

- A buzzabout API key. Generate one in **Settings → API keys** in the
  web app. Treat it like a password.
- Claude Desktop ([download](https://claude.ai/download)) or a Claude.ai
  workspace with custom integrations enabled.

## Add the server

<Steps>

<Step>

### Open the integration settings

- **Desktop:** `Settings → Developer → Edit Config`. Claude opens
  `claude_desktop_config.json` in your editor.
- **Web (claude.ai):** `Settings → Integrations → Add custom`.

</Step>

<Step>

### Paste the buzzabout block

<Tabs items={['Claude Desktop (JSON)', 'Claude.ai (HTTP / SSE)']}>
  <Tab value="Claude Desktop (JSON)">
    ```json title="claude_desktop_config.json"
    {
      "mcpServers": {
        "buzzabout": {
          "command": "npx",
          "args": [
            "-y",
            "@buzzabout/mcp-server"
          ],
          "env": {
            "BUZZABOUT_API_KEY": "YOUR_API_KEY"
          }
        }
      }
    }
    ```
  </Tab>
  <Tab value="Claude.ai (HTTP / SSE)">
    Use the hosted SSE endpoint — no local install required.

    | Field    | Value                                                |
    | -------- | ---------------------------------------------------- |
    | Name     | `buzzabout`                                          |
    | URL      | `https://mcp.buzzabout.ai/v1/sse`                    |
    | Auth     | `Authorization: Bearer YOUR_API_KEY` |
  </Tab>
</Tabs>

</Step>

<Step>

### Restart Claude

Desktop reloads the config on restart. Web takes effect on the next
conversation.

</Step>

<Step>

### Verify the connection

Open a new conversation and ask:

> *"Which buzzabout tools do you have available?"*

Claude should list `list_topics`, `create_topic`, `run_analysis`,
`fetch_mentions`, and `summarize_brief`. If it doesn't, see
[Troubleshooting](#troubleshooting) below.

</Step>

</Steps>

## A first prompt

Once the server is wired up, try this:

> *"Run a `pattern_match` analysis for the topic 'cold brew' on Reddit
> over the last 7 days. Then summarise the top three hooks and link the
> mentions you cite."*

Claude will:

1. Call `list_topics` to see whether *cold brew* already exists.
2. If not, call `create_topic`.
3. Call `run_analysis` with `kind: "pattern_match"` and a 7-day window.
4. Call `fetch_mentions` for the top patterns.
5. Stream a written summary back to you with citations.

Each tool call is visible in Claude's UI — click it to inspect what was
sent and what came back.

## Permissions and scopes

The API key you paste in the config inherits its workspace's
permissions. To restrict what Claude can do, generate a **scoped key**
under `Settings → API keys → Scoped key`:

| Scope                | What it allows                                              |
| -------------------- | ----------------------------------------------------------- |
| `topics:read`        | List existing topics. Required for the simplest workflows.  |
| `topics:write`       | Create / update topics.                                     |
| `analyses:read`      | Read finished analyses you've already kicked off.           |
| `analyses:write`     | Run new analyses against any readable topic.                |
| `mentions:read`      | Pull cited mentions for an analysis.                        |

A "read-only" key (`topics:read`, `analyses:read`, `mentions:read`) is
a safe default for casual exploration.

## Troubleshooting

<Callout type="warn" title="Server not showing up">
  **Desktop:** Confirm `claude_desktop_config.json` is valid JSON
  (Claude logs to `~/Library/Logs/Claude/main.log` on macOS). A common
  mistake is pasting the snippet outside an existing `mcpServers`
  block.

  **Web:** Make sure your workspace plan includes "Custom integrations"
  — Free / Pro accounts don't see the option, only Team / Enterprise.
</Callout>

If the tools list is empty, the most likely causes are:

1. **Bad API key** — the server returns 401 on every call. Re-issue and
   re-paste it.
2. **Outdated `@buzzabout/mcp-server`** — Claude pins the npm version on
   first install. Run `npx clear-npx-cache` and restart Claude.
3. **Network proxy / firewall** — the server needs outbound HTTPS to
   `api.buzzabout.ai`.

Still stuck? Open a thread in [the support channel](mailto:support@buzzabout.ai).
